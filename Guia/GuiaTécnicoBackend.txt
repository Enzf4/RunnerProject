URL da API: POST https://dcvmglbhuoahxvtsrjoz.supabase.co/auth/v1/signup

3. Banco de Dados (Tabelas)
Todas as tabelas estão protegidas por RLS. Os usuários só podem alterar o que lhes pertence.

Atualizar Perfil do Usuário (profiles)
Colunas: id, name, photo_url, pace_medio, cidade

URL da API: PATCH https://dcvmglbhuoahxvtsrjoz.supabase.co/rest/v1/profiles?id=eq.[ID_DO_USUARIO]

Headers exigidos na API: Authorization: Bearer [TOKEN_DO_USUARIO]

Como fazer no SDK (React):

4. Storage (Upload de Imagens)
O armazenamento de arquivos é separado do banco de dados. O fluxo é: 1) Fazer o upload do arquivo e pegar a URL gerada; 2) Fazer o UPDATE na tabela (profiles ou clubs) com essa URL.

Bucket: avatars (Fotos de Perfil)
O usuário deve salvar o arquivo com o próprio ID.

URL da API: POST https://dcvmglbhuoahxvtsrjoz.supabase.co/storage/v1/object/avatars/public/[ID_DO_USUARIO].png

Como fazer no SDK (React):

JavaScript
const { data, error } = await supabase.storage
  .from('avatars')
  .upload(public/${usuarioLogado.id}.png, arquivoDaImagem, { upsert: true })

// Para pegar a URL após o upload:
const publicUrl = supabase.storage.from('avatars').getPublicUrl(public/${usuarioLogado.id}.png).data.publicUrl
